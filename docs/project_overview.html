💡 What Is SQLOps?

The simplest explanation of our project

SQLOps is a training environment where an AI agent (like ChatGPT) learns to work with databases. Think of it as a "practice arena" — we give the AI a company database full of employees, departments, sales records, and projects. Then we ask it questions like "How many engineers do we have?" or "Fix this broken query." The AI writes SQL code, we check if it's correct, and give it a score from 0 to 1.

🎯 The 3 Tasks

Easy → Medium → Hard, each with 3 challenges scored 0.0 to 1.0

✅ Easy

Task 1: Data Retrieval

The AI is asked plain-English questions about the database and must write correct SQL queries to get answers.

  • 1
    "How many active employees are in the Engineering department?"
    Skills tested: COUNT, WHERE, JOIN
  • 2
    "List employees hired after 2024-01-01 — show full name and department"
    Skills tested: SELECT, JOIN, date filtering
  • 3
    "Total salary budget per department — highest first"
    Skills tested: GROUP BY, SUM, ORDER BY
⚠️ Medium

Task 2: Query Debugging

The AI receives broken SQL queries with bugs and must fix them to produce correct results. It also gets the expected output to compare against.

  • 1
    Wrong JOIN type — LEFT JOIN should be INNER JOIN, causing NULL rows
    Bug type: Logic error
  • 2
    Incorrect GROUP BY — Missing column in GROUP BY clause, wrong aggregate
    Bug type: Syntax + logic error
  • 3
    Subquery returns multiple rows — Correlated subquery used with = instead of IN
    Bug type: Runtime error
🔴 Hard

Task 3: Data Analysis

Complex analytical questions requiring window functions, CTEs, and multi-step reasoning. These are the kinds of queries senior data engineers write.

  • 1
    "Top 3 departments by average employee tenure (in years)"
    Skills: Date arithmetic, ranking, aggregation
  • 2
    "Month-over-month revenue growth rate for 2024-2025"
    Skills: Window functions (LAG), date grouping, percentage calc
  • 3
    "Employees earning above dept. average but below company median"
    Skills: CTEs, subqueries, statistical functions

📊 How Scoring Works

Partial credit system — not just right or wrong

Instead of binary pass/fail, we give partial credit based on how close the AI's answer is to the correct one. This gives the AI meaningful learning signals even when it's not perfect.

Score Scale

0.0
Error
0.15
Wrong cols
0.30
Right cols
0.50
50%+ match
0.75
80%+ match
1.0
Perfect
Score What It Means Example
0.0 Query fails to run or is completely wrong Syntax error, wrong table name
0.15 Query runs but returns wrong columns Asked for name & salary, got name & email
0.30 Correct columns but wrong data Right column names, but filter is wrong
0.50 Partially correct — 50%+ rows match Got some results right, missed others
0.75 Mostly correct — 80%+ rows match Almost perfect, minor ordering issue
1.0 Perfect match! All rows, columns, and order are correct

🧠 Why partial credit matters: In reinforcement learning, the AI needs gradient signals to improve. If we only said "right or wrong," the AI would struggle to learn. Partial credit tells the AI "you're getting warmer" — which is crucial for efficient learning.

Final Score Calculation

For each task:

# Each task has 3 challenges, each worth 0.0 to 1.0
total_reward = challenge_1_score + challenge_2_score + challenge_3_score
max_possible = 3.0
final_score = total_reward / max_possible   # Clamped to [0.0, 1.0]

🔧 The AI's Tools

5 MCP tools the AI agent can use to interact with the environment

The AI doesn't just send random messages. It has specific tools it can call, just like a human developer using a database client:

🔍 get_schema()

Returns the full database schema — all CREATE TABLE statements. The AI reads this to understand what tables and columns exist.

📋 get_challenge()

Returns the current challenge — the question to answer, any context (like broken SQL for debugging tasks), and what format the answer should be in.

▶️ execute_sql(query)

Runs a SQL query and returns results — for exploration only, no grading. The AI can use this to test ideas before submitting.

✅ submit_answer(query)

Submits the AI's final SQL answer. The grader runs it, compares results to the expected answer, and returns a score (0.0–1.0).

📈 get_progress()

Shows how far along the AI is — challenges completed, total reward earned, challenges remaining.

Example Interaction (Step by Step)

# 1. Environment starts up
result = await env.reset() # → "Ready! Task: data_retrieval"

# 2. AI reads the database structure
schema = await env.call_tool("get_schema")
# → CREATE TABLE employees (...), CREATE TABLE departments (...)

# 3. AI reads the first question
challenge = await env.call_tool("get_challenge")
# → "How many active employees are in Engineering?"

# 4. AI tests a query (optional)
test = await env.call_tool("execute_sql",
    query="SELECT * FROM departments LIMIT 3")

# 5. AI submits its answer
result = await env.call_tool("submit_answer",
    query="SELECT COUNT(*) FROM employees e JOIN departments d ON e.department_id = d.id WHERE d.name = 'Engineering' AND e.is_active = 1")
# → reward: 1.0 ✓ Perfect!

🚀 Deployment Plan

Docker container → Hugging Face Spaces → Ready for judging

Step 1: Docker Container

We package the entire environment into a Docker container — a self-contained box that runs anywhere. It includes:

  • Python 3.11 + all dependencies
  • FastAPI server (handles HTTP + WebSocket)
  • SQLite database (created fresh on reset)
  • All task/challenge definitions
  • Grading engine
# Build the container
docker build -t sqlops-env ./server

# Run it locally
docker run -p 8000:8000 sqlops-env

# Test it's working
curl http://localhost:8000/health # → {"status": "ok"}

Step 2: Deploy to Hugging Face Spaces

Using the OpenEnv CLI, we push the environment to Hugging Face Spaces — a free hosting platform:

# One command to deploy
openenv push --repo-id your-username/sqlops-env
